# Learn Open Source - 开源源码学习课程生成器 (Simple but Effective)

一个能够系统学习开源项目的skill，将任意开源项目拆解为结构化的Markdown课程文件，帮助开发者系统地、逐步地深入理解项目源码。

持续更新ing..... 后续会逐步补全Scripts与Reference

## 适用场景

- 想学习某个开源项目但不知从何入手
- 需要快速理解项目的整体架构和调用链路
- 希望以课程形式沉淀源码阅读笔记
- 团队内部技术分享或新人培训

## 支持的项目类型

- GitHub URL 或本地路径
- 任何编程语言（Go / Rust / Python / JavaScript / Java 等）
- 任何项目类型（CLI 工具、Web 服务、框架、SDK 等）

## 工作流程

技能按五个阶段顺序执行，不跳过任何步骤：

| 阶段 | 说明 |
|------|------|
| 阶段一：项目发现 | 克隆项目、生成文件树、识别技术栈、定位入口文件 |
| 阶段二：架构追踪 | 从入口出发逐层追踪调用链路，画出完整架构图 |
| 阶段三：课程规划 | 按模块拆分章节，生成目录供用户确认 |
| 阶段四：逐章深挖 | 逐文件、逐方法深入分析职责、签名、核心逻辑 |
| 阶段五：生成课程 | 将所有分析整理为 Markdown 文件输出到 Tutorial/ 目录 |

## 输出结构

```
<项目根目录>/Tutorial/
├── 00-课程大纲.md              # 架构图 + 学习路线
├── 01-项目概览与开发环境.md      # 技术栈、目录结构、如何跑起来
├── 02-入口与启动流程.md
├── 03-路由与中间件.md
├── 04-核心业务逻辑.md
├── ...
└── 0N-总结与扩展.md            # 设计模式、可扩展点
```

## 使用方式

在对话中，直接表达学习意图即可触发该技能：

```
帮我学习 https://github.com/gin-gonic/gin 这个项目
```

```
分析一下这个项目的源码：/path/to/local/project
```

## 核心特点

- **从入口到出口**：完整追踪调用链，画出从程序入口到终点的架构图
- **逐方法讲解**：每个关键方法都解释目的、参数、返回值和核心逻辑
- **模块化组织**：按功能拆分章节，每章聚焦一个主题，覆盖 3-8 个紧密相关的文件
- **深度优先**：核心模块的每个重要方法都讲到，不浮于表面
- **Mermaid 架构图**：整体架构图和每课局部调用图均为强制要求
