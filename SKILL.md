# 开源源码学习课程生成器

把开源项目拆解为结构化 Markdown 课程，帮助用户系统理解源码。

## 核心原则

- **从入口到出口**：追踪完整调用链，画出从入口到终点的架构图
- **逐方法讲解**：每个关键方法解释目的、参数、返回值和核心逻辑
- **模块化组织**：按模块/功能拆分章节，每章聚焦一个主题
- **深度优先**：核心模块的每个重要方法都要讲到，宁可深入不要浮于表面

## Reference Map

所有课程模板和分析指南独立为 reference 文件，按需加载：

| 文件 | 内容 | 何时加载 |
|------|------|----------|
| `references/language-analysis-guides.md` | 6 种编程语言的分析切入点和关注点 | 阶段一 — 识别语言后加载对应部分 |
| `references/architecture-patterns.md` | 6 种架构模式的 Mermaid 图模板 | 阶段二 — 画架构图时参考 |
| `references/course-templates.md` | 课程 Markdown 模板和目录结构 | 阶段五 — 生成文件时使用 |
| `references/mermaid-diagram-guide.md` | Mermaid 图类型选择与绘制指南 | 阶段二/四 — 需要画图时参考 |
| `references/quality-standards.md` | 深度标准、反模式清单、阶段自检 | 全部阶段 — 开始前读取，每阶段结束自检 |

## 工作流程

按以下五个阶段顺序执行。每阶段完成后简短汇报进展，然后继续。**不要跳过任何阶段。**

开始前先读取 `references/quality-standards.md`，了解质量标准和反模式。

### 阶段一：项目发现

1. GitHub URL → `gh repo clone` 克隆到本地；本地路径则直接使用
2. 生成完整文件树（排除 node_modules、.git、vendor、target、dist、build 等）
3. 识别项目类型：编程语言、框架、构建系统
4. 阅读 README、package.json/Cargo.toml/go.mod/pom.xml，理解项目功能
5. 找到所有入口：main 函数、CLI 入口、HTTP 启动点、导出主模块

> **按需加载**：识别语言后，读取 `references/language-analysis-guides.md` 中对应语言的分析策略。

汇报格式：
```
## 项目概览
- 项目名: xxx
- 语言/框架: xxx
- 核心功能: xxx（一句话）
- 入口文件: xxx
```

### 阶段二：架构追踪（最关键）

1. **从入口出发**：阅读入口文件，理解程序启动流程
2. **逐层追踪**：顺 import/include/require 关系向下追，直到业务终点
3. **画架构图**：用 Mermaid `graph TD` 画出从入口到出口的完整调用链路。必须包含入口点、中间层（中间件/拦截器）、业务逻辑层、数据层、出口点

> **按需加载**：读取 `references/architecture-patterns.md`，选择匹配的架构模式；读取 `references/mermaid-diagram-guide.md`，选择合适图类型。

4. **识别核心模块**：标注业务逻辑集中的核心模块 vs 工具/配置辅助模块

架构图放入 `00-课程大纲.md`。

### 阶段三：课程规划

基于架构分析拆解课程章节：

- **第 0 章**：课程大纲（架构图、学习路线、章节概览）
- **第 1 章**：项目概览与开发环境（跑起来、技术栈、目录结构总览）
- **第 2 章起**：按调用链路组织（入口 → 中间层 → 核心业务 → 数据层 → 出口）
- **最后一章**：总结与扩展（设计模式总结、可扩展点、下一步建议）

每章覆盖 3-8 个紧密相关的文件。

**生成章节目录后向用户确认**，确认后再进入阶段四。

### 阶段四：逐章深挖

对每章的每个文件进行深度分析：

1. **文件职责概述**：在整体架构中的位置和作用
2. **关键方法详解**：每个公开方法的签名、目的、核心逻辑（解释思路不是翻译代码）、调用关系
3. **设计要点**：设计模式、技巧、潜在坑点

**不要跳过任何文件**。核心模块的 private 方法也要讲解。配置/工具文件至少写 1 段概述。

### 阶段五：生成课程文件

> **按需加载**：读取 `references/course-templates.md`，使用标准模板生成所有文件。

输出到**目标项目根目录**下的 `Tutorial/` 文件夹：

```
<项目根目录>/Tutorial/
├── 00-课程大纲.md
├── 01-项目概览与开发环境.md
├── 02-入口与启动流程.md
├── ...
└── 0N-总结与扩展.md
```

每个阶段结束时，用 `references/quality-standards.md` 的检查清单自检。

## 重要规则

1. 每个文件都要覆盖，核心模块的每个方法都要讲解
2. 严格按五个阶段执行，架构追踪不能省略
3. 课程大纲的整体架构图和每课的局部调用图是 Mermaid 强制要求
4. 代码引用从实际文件复制，带上准确行号
5. 根据项目语言调整分析方式（详见 `references/language-analysis-guides.md`）
6. 所有课程文件输出到 `<项目根目录>/Tutorial/`
