# 课程文件模板

所有生成的课程 Markdown 文件使用以下统一模板。

---

## 第 0 课模板：课程大纲

```markdown
# 课程大纲：[项目名] 源码解析

## 项目简介
一句话描述 + 技术栈说明。

## 整体架构图
（Mermaid 图，从入口到出口的完整调用链路）

## 学习路线
| 章节 | 标题 | 核心内容 | 文件数 |
|------|------|----------|--------|
| 01 | xxx | xxx | N |
| 02 | xxx | xxx | N |
| ... | ... | ... | ... |

## 阅读建议
- 建议按章节顺序阅读，不要跳章
- 每章末尾的思考题建议先自己想一想再看下一章
- 建议边看边在 IDE 中打开对应源文件对照阅读
```

**字段要求**：
- **项目简介**：1-2 句话，说清楚项目做什么 + 技术栈
- **整体架构图**：Mermaid `graph TD`，必须覆盖入口 → 中间层 → 数据层 → 出口
- **学习路线表**：每行对应一章，按调用链路顺序排列
- **文件数**：每章覆盖 3-8 个源文件

---

## 标准课程章节模板（第 1 课起）

```markdown
# 第X课：[课程标题]

## 本课目标
简要说明本课要搞懂的 2-4 个核心问题。

## 涉及的源文件
| 文件 | 作用 |
|------|------|
| `src/xxx.go` | 负责 xxx |
| `src/yyy.go` | 负责 yyy |

## 调用关系图（局部）
用 Mermaid 画出本章涉及模块之间的调用关系。

## 文件详解

### `src/xxx.go`

#### 概述
这个文件负责 xxx，在整体架构中处于 xxx 层。

#### 方法详解

##### `func DoSomething(param Type) ReturnType`
- **目的**：xxx
- **参数**：param - xxx
- **返回值**：xxx
- **核心逻辑**：
  1. 首先 xxx
  2. 然后 xxx
  3. 最后 xxx

（关键代码片段 + 注释）

##### `func AnotherMethod()`
...逐个方法重复...

---

### `src/yyy.go`

...重复相同结构...

## 本课小结
- 要点 1
- 要点 2
- 抛出 1-2 个引导下一课的思考题
```

**字段要求**：
- **本课目标**：不要列"学习 X 文件"，要写"理解 A 如何调用 B""搞懂 C 的设计模式"这类有洞察的问题
- **调用关系图**：只画本章涉及的文件之间的调用关系，不必画完整架构
- **文件详解**：按调用顺序排列，被调用的在前；互相没有依赖的按 alphabet 排
- **核心逻辑**：解释设计思路，不要逐行翻译代码
- **思考题**：自然引出下一课的内容，让章节之间有连贯性

---

## 输出目录结构

所有课程文件输出到目标项目根目录下的 `Tutorial/` 文件夹：

```
<目标项目根目录>/Tutorial/
├── 00-课程大纲.md
├── 01-项目概览与开发环境.md
├── 02-入口与启动流程.md
├── 03-路由与中间件.md
├── 04-核心业务逻辑.md
├── ...
└── 0N-总结与扩展.md
```

章节数量取决于项目复杂度。小型项目（<20 个源文件）通常 5-7 章，中型项目（20-50 个源文件）8-12 章，大型项目需拆分更多。
