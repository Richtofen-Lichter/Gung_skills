# 课程质量标准与自检清单

确保课程输出达到"能真正帮助理解源码"的深度和覆盖度。

---

## 深度标准

每个核心方法的讲解必须达到"三层理解"，缺一不可：

| 层级 | 要求 | 反例 |
|------|------|------|
| **做了什么** | 描述方法的输入处理步骤和输出 | "这个方法接收一个 int，返回一个 string" |
| **怎么做的** | 解释算法的关键步骤和分支逻辑 | 逐行翻译代码（应解释思路，不是翻译语法） |
| **为什么这样设计** | 解释设计决策、权衡、上下文 | 省略（这层最容易缺失，但最关键） |

**检查**：随机挑一个核心方法，问自己——读者看完能回答"为什么不直接传参数而要封装成 struct""为什么用 channel 而不是 mutex"吗？如果不能，深度不够。

---

## 覆盖面标准

| 文件类型 | 最低要求 |
|----------|---------|
| 核心业务文件（逻辑密集） | **每个公开方法**都要讲解；关键 private 方法也要讲 |
| 辅助文件（配置、工具） | 至少 1 段概述，讲清楚它服务于谁、被谁调用 |
| 入口文件 | 详细分析启动/初始化流程的每一步 |
| 测试文件 | 无需逐文件分析；在相关实现文件的讲解中引用关键测试 |

**检查**：打开项目的文件列表，逐文件对照——有没有文件被完全跳过？配置文件和常量文件至少要有 1 句话说明。

## 反模式清单

以下 6 种写法必须避免：

### 反模式 1：纯翻译代码
```
❌ "它先创建一个 map，然后遍历 slice，把每个元素的 ID 作为 key 放入 map，最后返回 map。"
✅ "将列表转为 HashMap，使后续查重操作从 O(n) 降到 O(1)。这是为了在下游的去重逻辑中快速判断元素是否已处理。"
```

### 反模式 2：跳过"简单文件"
```
❌ "config.go 是配置文件，没什么好说的，跳过。"
✅ "config.go 通过 Viper 从 YAML 文件和环境变量加载配置。它导出一个全局单例 Config 对象，
被所有模块引用。值得注意的设计是：配置变更不热加载，需要重启服务。"
```

### 反模式 3：只讲"是什么"不讲"为什么"
```
❌ "这里使用了一个 interface 来抽象 LLM Provider。"
✅ "使用 LLMProvider interface 抽象所有 LLM 后端，因为系统需要支持 OpenAI / Ollama / Azure
三种后端切换。如果硬编码 OpenAI SDK，每次换后端都要改业务代码。Factory 模式 + 接口让
组件切换变成纯配置变更。"
```

### 反模式 4：Mermaid 图过于简单
```
❌ graph TD
    A[main] --> B[handler]
    B --> C[response]
```
缺少中间件层、数据层、错误处理路径。正确的架构图必须包含入口 → 中间层 → 业务层 → 数据层 → 出口的完整链路。

### 反模式 5：代码引用不准
```
❌ "该函数大致长这样：func foo(x int) { ... }"（凭记忆写，参数类型可能错误）
✅ 从实际文件 copy-paste 代码，带上准确的行号引用
```

### 反模式 6：忽略错误处理路径
```
❌ 只讲"正常情况下的数据流"
✅ 也讲"出错时：重试机制、fallback 策略、错误包裹传递"
```

## 阶段完成自检

每个阶段结束时，用以下问题自检：

### 阶段一（项目发现）
- [ ] 文件树是否排除了 node_modules/.git/vendor/target/dist/build？
- [ ] 是否阅读了 README 和 package.json/go.mod/Cargo.toml？
- [ ] 是否找到了所有入口文件（不只是一个 main.go，可能有 CLI 入口、HTTP 入口、Worker 入口）？

### 阶段二（架构追踪）
- [ ] 是否从入口文件一路追到了终端（HTTP 响应 / CLI 输出 / 文件写入）？
- [ ] 调用链路是否完整？有没有遗漏的中间件、拦截器、hook？
- [ ] 架构 Mermaid 图是否包含入口 → 中间层 → 业务层 → 数据层 → 出口？
- [ ] 是否标注了哪些是核心模块、哪些是辅助模块？

### 阶段三（课程规划）
- [ ] 章节是否按调用链路顺序排列（入口 → 中间 → 核心 → 数据 → 出口）？
- [ ] 每章覆盖 3-8 个文件？没有"一章一个简单文件"也没有"一章塞 20 个文件"？
- [ ] 用户确认了章节目录？

### 阶段四（逐章深挖）
- [ ] 每个文件的每个公开方法都讲解了吗？
- [ ] 核心模块的 private 方法也覆盖了吗？
- [ ] 每个核心方法的讲解达到了"三层理解"吗？

### 阶段五（生成课程文件）
- [ ] 所有文件输出到了 `<项目根目录>/Tutorial/` 吗？
- [ ] 课程大纲包含 Mermaid 架构图吗？
- [ ] 每课都包含局部调用关系 Mermaid 图吗？
- [ ] 代码引用是直接从实际文件 copy 的吗？
